# Guide de contribution

Nous sommes ravis que vous souhaitiez contribuer à ce projet !  
**Toutes les contributions, rapports de bogues, corrections de bogues, améliorations de la documentation, améliorations et idées sont les bienvenues.**

## Comment contribuer

1. **Forkez** le dépôt sur votre compte GitHub.
2. **Clonez** le fork en local :  
   `git clone https://github.com/<votre-utilisateur>/<nom-du-depot>.git`
3. **Créez une branche** pour votre contribution :  
   `git checkout -b amelioration/ma-fonctionnalite`
4. **Effectuez vos modifications** en respectant le style de code du projet.
5. **Testez** soigneusement vos changements.
6. **Commitez** avec un message clair :  
   `git commit -m "Description concise de la modification"`
7. **Poussez** la branche :  
   `git push origin amelioration/ma-fonctionnalite`
8. **Ouvrez une Pull Request** depuis votre branche vers la branche `main` du dépôt d'origine.

## Bonnes pratiques

- **Signalement de bugs** : ouvrez une issue en décrivant le problème, les étapes pour le reproduire et l'environnement utilisé.
- **Propositions d'améliorations** : détaillez votre idée et son intérêt pour le projet.
- **Code** : commentez votre code, suivez la convention de nommage en place et assurez-vous qu'il n'introduit pas de régressions.
- **Documentation** : mettez à jour le fichier `README.md` ou toute documentation pertinente si nécessaire.

## Processus de revue

Toutes les Pull Requests sont examinées par les mainteneurs du projet. Nous pouvons demander des ajustements avant la fusion. Merci de rester courtois et constructif dans les échanges.

Nous avons hâte de découvrir vos contributions !
